Расширение права SUID, GUID, stricky bit
Бит SUID позволяет выполнить файл от имени владельца файла
Эта возможность используется для предоставления дополнительных прав процессу,тогда необходимо обращение к недоступным для обычных пользователей ресурсам.
Установка бита осуществляется командой chmod:

sudo chmod u+s <имя файла>
авильно.
Классов Александр
sudo chmod 4775 <имя файла>

Установленный бит отображается буквой s в разрешениях запуска для владельцев файла:
Доступ: (4755.-rwsr-xr-x)

Бит SUID позволяет выполнить файл от имени владельца файла
Эта возможность используется для предоставления дополнительных прав процессу,тогда необходимо обращение к недоступным для обычных пользователей ресурсам.
Установка бита осуществляется командой chmod:

sudo chmod u+s <имя файла>
авильно.
Классов Александр
sudo chmod 4775 <имя файла>

Установленный бит отображается буквой s в разрешениях запуска для владельцев файла:
Доступ: (4755.-rwsr-xr-x)

Если владелец не имеет прав доступа к выполнению файла, установленный бит отображается прописной буквой s
Доступ: (4655/-rwsr-xr-x)
SUID имеет смысл устанавливать ждя файлов,но не для папок.

Бит GUID позволяет запускать файл от имени группы владельцев, а также может влиять на режимы доступа от объекта, находящегося внутри папки с установленным битом GUID

Установка бита осуществляется командой chmod:

sudo chmod g+s <имя файла или имя папки>
sudo chmod 2775 <имя файла или имя папки>

При установке GUID на папку все объекты, созданные после этого внутри этой папки, наследуют группу владельцев

Установленный бит отображается буквой s в разрешениях для запуска для группы владельцев:

Доступ: (2776/-rwxrwsr--) - для файла
Доступ: (2775/drwxrwsr-x) - для папки

Каталоги внутри папки с установленным GUID-битом получают такую же группу владельцев и установленный GUID-бит
